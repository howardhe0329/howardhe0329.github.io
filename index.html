Soul API网关使用入门

Soul介绍

soul是基于WebFlux实现的响应式的API，具有异步、高性能、跨语言等特点。

Soul主要功能

  • 支持各种语言，无縫集成到Dubbo、Spring Cloud、Spring Boot中。
  • 支持各种语言(http协议)，支持 dubbo，springcloud协议。
  • 灵活的流量筛选，能满足各种流量控制。
  • 内置丰富的插件支持，鉴权，限流，熔断，防火墙等等。
  • 流量配置动态化，性能极高，网关消耗在 1~2ms。
  • 支持集群部署，支持 A/B Test, 蓝绿发布。

Soul依赖环境

依赖环境：

  • jdk 1.8+
  • Maven 3.6.0+
  • MySQL 5.7+

开发工具：

  • IntelliJ IDEA

Soul Admin数据库设计

plugin表

插件

字段 描述
id 主键
name 插件名称
config 配置信息，JSON Format。如dubbo插件，获取dubbo元数据信息可以通过配置中心获取{“register”:”zookeeper://localhost:2181”}
role 角色，指定哪个角色可以修改插件
enabled 是否开启插件

plugin_handle表

插件处理配置

字段 描述
id 主键
plugin_id 插件id
field 属性
label 标签
data_type 数据类型 1:number;2:string
type 类型 1:selector;2:rule;3:plugin
sort 顺序
ext_obj 扩展配置信息 JSON FORMAT

selector表

选择器

字段 描述
id 主键
plugin_id 插件ID
name 选择器名称
match_mode 匹配模式 0:and;1:or
type 类型 0:full flow;1:custom flow
sort 顺序
handle 处理逻辑，不同的插件有不同的规则配置。JSON Format
enabled 是否开启
loged 是否输出日启
continued 是否执行后续选择器

selector_condition表

选择器条件

字段 描述
id 主键
selector_id 选择器ID
param_type 参数类型 如: uri、query、post、host、ip、header等等
operator 操作符 如：=、like、match、regEx等
param_name 参数名称
param_value 参数值

rule表

规则

字段 描述
id 主键
selector_id 选择器ID
match_mode 匹配模式 0:and;1:or
name 名称
enabled 是否开启
loged 是否输出日志
sort 顺序
handle 处理逻辑，不同的插件有不同的规则配置。JSON Format

rule_condition表

规则条件

字段 描述
id 主键
rule_id 规则ID
param_type 参数类型 如: uri、query、post、host、ip、header等等
operator 操作符 如：=、like、match、regEx等
param_name 参数名称
param_value 参数值

meta_data表

元数据 好像只有dubbo相关配置元数据

字段 描述
id 主键
app_name Application name
path 路径，唯一
path_desc 路径描述信息
rpc_type RPC类型 如：dubbo
service_name Service Name 接口名称
method_name Method Name 方法名称
parameter_types 参数类型
rpc_ext 扩展信息
enabled 是否开启

soul_dict表

字典表 一般用来配置流量控制、熔断策略、权限等配置

字段 描述
id 主键
type 类型
dict_code Dictionary encoding
dict_name dictionary name
dict_value dictionary value
desc dictionary description
sort 顺序
enabled 是否开启
